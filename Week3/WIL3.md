# __:pushpin: 필수 정리 내용__
## __1. Spring Bean__
Spring 컨테이너에 의해 관리되는 자바 객체를 의미한다. Spring 컨테이너는 Spring Bean의 생명 주기를 관리하고 생성된 Spring Bean에 추가적인 기능을 제공한다.

## __2. JPA(Java Persistence API)__
자바의 ORM을 위한 표준 기술로 Hibernate, Spring JPA, EcliplseLink 등과 같은 구현체가 있다.
### __2.1. ORM(Object-Relational Mapping)__
자바의 객체와 관계형 DB를 맵핑하는 것으로 DB의 특정 테이블이 자바의 객체로 맵핑되어 SQL문을 일일이 작성하지 않고 객체로 구현할 수 있도록 하는 프레임워크를 말한다.
### __2.2. 장점__
* 쿼리를 일일이 작성할 필요가 없어 작성해야 하는 코드가 적음
* 객체 위주로 코드를 작성하기 때문에 가독성이 좋고 수정이 용이함
* 쿼리를 직접 작성하는 것이 아니라 DB벤더에서 독립적인 개발 가능

## __1. 의존 관계__
A가 B를 의존한다는 말은 B의 기능이 추가되거나 변경되면 그 영향이 A에 미치는 것을 의미한다.<br/>ex) 멤버 컨트롤러는 멤버 서비스를 통해 회원가입을 처리하고 데이터를 조회하는데 이를 멤버 컨트롤러가 멤버 서비스를 의존한다고 표현할 수 있다.
### __1.1. Autowired__
필요한 의존 객체의 타입에 해당하는 bean을 찾아 주입한다. Autowired의 기본값은 true이기 때문에 의존성을 주입할 대상을 찾지 못한다면 애플리케이션 구동에 실패한다.

## __2. Spring Data JPA__
Spring 프레임워크에서 JPA를 편리하게 사용할 수 있도록 지원하는 프로젝트이다.
* CRUD 처리를 위한 공통 인터페이스 제공
* 리포지토리 개발 시 인터페이스만 작성하면 실행 시점에 스프링 데이터 JPA가 구현 객체를 동적으로 생성해서 주입
* 데이터 접근 계층을 개발할 때 구현 클래스 없이 인터페이스만 작성해도 개발을 완료할 수 있도록 지원
### __2.1. CRUD__
컴퓨터 소프트웨어가 가지는 기본적인 데이터 처리 기능인 Create(생성), Read(읽기), Update(갱신), Delete(삭제)를 의미한다.
